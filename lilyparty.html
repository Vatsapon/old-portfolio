Lily Party is a 2D multiplayer top-down shooter game where players need to survive together through a wave of monsters until they're all eliminated. But players also need to compete with each other to gain more power through upgrades and points to gain the top ranking at the end of the game.

The concept of the game was mainly inspired by Lovers in a Dangerous Spacetime with a similar genre. The game mainly focuses on combat which combines cooperation and competition to fight against enemies and compete with each other.

How It Works & Progress

Since I had little multiplayer experience, this was my first multiplayer project and it turned out to be one of my most challenging ones. For this to work with multiple inputs and separate controls for each player, I will need to conduct some research to understand a new input system.

I made a decision to divide players across two systems. The first is the Player Ship, in which each player controls the weapon and thruster. The second is Player Body, which includes all of the player's ships and acts as a single ship with collision detection and health. Compared to making solely the Player Body system, this will be easier to manage. Since the system was similar to the last project, I applied the Factory design pattern as the structure for the Enemies.

For each ship, I decided to program the manager to each Player Ship using the Decorator design pattern for weapons and upgrades. We can separate in the same way to run tests as well. Furthermore, it is better structured so that each class may manage its system independently of each other.

My Tool Creation - Input Display

The Input Displayer is one of the tools I made for this project. It will display an input icon based on the type of controller and input keys.

Keyboard users, for instance, will use WASD keyboard icons. Controller icons such as the D-Pad and Trigger Btn are used by controller users. This helps with the integration of HUD/UI for tutorial instruction, control maps, and other usage for programmers and designers users.

It functions by first creating a ScriptableObject and assigning an icon key sprite sheet as a variable. Data will then be generated automatically. There will be a listener button to manually set the key path and name based on incoming user input if any of them are generated incorrectly. The key icon will then be displayed in the ControlKey component depending on the active control.
