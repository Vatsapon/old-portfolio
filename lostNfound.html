Lost & Found Co. is a 2D hidden object with a narrative game. The player played as Ducky, the first employee of a startup company founded by Mei, a long-forgotten goddess. The player must help Ducky find lost items in certain areas.

The concept of the game is based on a classic hidden game where the player needs to find objects on the screen. Except that this game has a light-hearted narrative and art style with unique gameplay that has more interaction with the game.

External Work - Porting from PC to Mobile

In addition to working on this project, which is internal, I had the opportunity to work on an external project that involved porting. One of my responsibilities is to migrate the input control system from the PC (Keyboard & Mouse) to the mobile device (Touch screen / Game Controller). This is a crucial task that must be completed in order to switch to a new input system.

Gesture or Conditional Input, including Double-Tap, Drag, and Swipe, is one of the most challenging aspects of creating touch screen controls. Thus, I made use of the Unity package's On-Screen Button system, which could only handle one action or input path at a time. However, because to the button mapping design's requirement for various actions with only one button such as Double-Tap for Rolling and Tap for Jumping.

I made the decision to look into and tweak the script on my own in order to create a new On-Screen Button system that allows for multiple actions/input paths. The button's input path, which simulates an input key, is how it performs. Additionally, data will be sent to the Event System as a triggering input key when the button is pressed. I made the decision to manually send data by adding an additional path when executing an event. When the condition is met, I can afterwards create a tap system that handles tap events and sends data.
