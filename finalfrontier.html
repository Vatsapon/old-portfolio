Final Frontier is a 2D top-down shooter game where you have to fight the enemies which trying to invade your base. The game concept is based on the Tower Defense game style but in free-form movement.

The concept is similar to other Tower Defense games. But the difference is that the player doesn't control each tower, but as a character to fight against incoming hostile enemies instead. The enemy appears in waves approaching from the edge of the map which the player can prepare and arm themselves in-between waves to fight against them.

The player can purchase and customize weapons and gadgets to gain stronger weapons with different attributes and stats after the end of each wave. The longer the wave, the stronger they are.

The deadline for completing this academic project was set for the full semester, or about 14 to 15 weeks. The goal of this project is to create a game containing at least fifty expression or fantasy elements. Fieldrunners and other 2D shooting games served as my inspiration for this project as a tower defense and free-form shooter game hybrid.

Therefore, I realized that most of my scripts are reusable for other projects and can be tweaked in more complex methods to improve performance. Examples of this include the Player Movement Script and Audio Manager, which can be made functional with only a few variables and a simple drag-and-drop action.

Even so, it's challenging to deal with the design and graphics while simultaneously working on other projects. However, it was one of the most challenging circumstances for me to grow as an individual and learn new skills.

My Tool Creation - Editor & Debug Tools

Most of the scripts I wrote for this project have customizable visual displays on the inspector that are easy to configure or modify using customization variables. This will boost productivity and enhance the visual appearance of the project, making it simpler to find, organize, and make changes efficiently.

I've created three different kinds of debugs for this project. One is the Editor, which modifies the entire inspector visualization. The second is an attribute, which modifies a certain property's appearance depending on a kind or condition of the variable. The third window is the Editor Window, which is a separate Inspector and fully customizable which can show any type of data.

This will help with debugging and data inspection during runtime for various purposes, including improvisation, balancing, and debugging.
